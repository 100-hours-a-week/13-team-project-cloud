# Contributing Guide

## Issue 관리

### Issue 템플릿

| 템플릿 | 용도 | 기본 라벨 | 자동 라벨링 |
|--------|------|----------|------------|
| Sprint Backlog | 스프린트 백로그 항목 | `백로그` | 우선순위 |
| Bug Report | 버그/오류 리포트 | `버그` | 심각도 |
| Incident Report | 장애/인시던트 기록 | `장애` | 심각도 |
| Feature Request | 기능 요청/개선 제안 | `기능요청` | 우선순위 |
| Question | 질문/논의 | `질문` | - |

### 라벨

#### 기본 라벨

| 라벨 | 설명 |
|------|------|
| `백로그` | 스프린트 백로그 항목 - 주간 과제 및 작업 단위 |
| `버그` | 버그/오류 - 기능이 의도대로 동작하지 않는 문제 |
| `장애` | 서비스 장애/인시던트 - 긴급 대응 필요한 운영 이슈 |
| `기능요청` | 기능 요청/개선 - 새로운 기능 추가 또는 기존 기능 개선 제안 |
| `질문` | 질문 및 논의 - 기술적 질문이나 의사결정 필요한 사항 |

#### 우선순위 라벨 (자동)

Sprint Backlog, Feature Request 템플릿에서 우선순위 선택 시 자동 적용

| 라벨 | 설명 |
|------|------|
| `우선순위:높음` | 긴급 - 즉시 처리 필요, 다른 작업보다 먼저 수행 |
| `우선순위:중간` | 보통 - 이번 스프린트 내 처리 권장 |
| `우선순위:낮음` | 여유 - 시간 여유 있을 때 처리해도 무방 |

#### 심각도 라벨 (자동)

Bug Report, Incident Report 템플릿에서 심각도 선택 시 자동 적용

| 라벨 | 설명 |
|------|------|
| `심각도:Critical` | 서비스 전체 중단 |
| `심각도:High` | 주요 기능 장애 |
| `심각도:Medium` | 일부 기능 장애 |
| `심각도:Low` | 경미한 영향 |

### Milestone (스프린트 관리)

주차별 Milestone을 사용하여 스프린트를 관리합니다.

1. **Milestone 생성**: `Week 1`, `Week 2` 등 주차별로 생성
2. **Issue 연결**: 백로그 이슈 생성 시 해당 주차 Milestone 선택
3. **진행률 확인**: Milestone 페이지에서 전체 진행률 확인

### Issue 작성 규칙

1. 적절한 템플릿 선택
2. 제목은 명확하게 작성
3. 필수 항목 모두 작성
4. 해당하는 Milestone 연결
5. 담당자(Assignee) 지정
